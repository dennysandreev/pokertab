# Poker Score Popup — описание для Codex

## Задача

Добавить в экран лидерборда информационный popup / modal, который объясняет пользователю, из чего складывается показатель **Poker Score**.

Popup должен открываться по нажатию на иконку `i` / `?` рядом с названием показателя **Poker Score** в лидерборде и в профиле игрока.

---

## Где показывать

### 1. Leaderboard

Рядом с заголовком или метрикой:

```text
Poker Score 84 ⓘ
```

При нажатии на `ⓘ` открывается popup.

### 2. Player Profile

В блоке статистики игрока:

```text
Poker Score
84 ⓘ
```

При нажатии открывается тот же popup.

## Блоки внутри popup

### 1. ROI Score

```text
ROI Score — эффективность игры

Показывает, насколько эффективно игрок превращает закупы в прибыль.

Формула:
ROI = чистая прибыль / сумма всех закупов × 100%

Чем выше ROI, тем лучше игрок использует свои закупы.
```

Вес:

```text
Вес в Poker Score: 40%
```

---

### 2. Win Rate

```text
Win Rate — процент плюсовых игр

Показывает, как часто игрок заканчивает игру в плюс.

Формула:
Win Rate = количество плюсовых игр / общее количество игр × 100%

Этот показатель отражает регулярность положительных результатов.
```

Вес:

```text
Вес в Poker Score: 30%
```

---

### 3. Stability Score

```text
Stability Score — стабильность игры

Показывает, насколько часто игрок избегает сильных минусовых сессий.

В MVP игра считается стабильной, если результат игрока не хуже, чем минус один ребай.

Например, если ребай 1 000 ₽, то результат -500 ₽ считается стабильным, а -2 500 ₽ — нет.
```

Вес:

```text
Вес в Poker Score: 20%
```

---

### 4. Volume Confidence

```text
Volume Confidence — доверие к статистике

Показывает, насколько можно доверять статистике игрока с учетом количества сыгранных игр.

Чем больше игр сыграл пользователь, тем выше доверие к его показателям.
```

Пример шкалы:

```text
1–2 игры: 20
3–5 игр: 50
6–10 игр: 75
10+ игр: 100
```

Вес:

```text
Вес в Poker Score: 10%
```

## Финальный поясняющий текст

```text
Poker Score нужен, чтобы рейтинг был честнее. Он не дает игроку с одной удачной игрой сразу стать первым и не оценивает только общий выигрыш.

Total Profit показывает, сколько игрок выиграл или проиграл в деньгах.
```

## UX-требования

1. Popup должен быть компактным и читабельным на экране телефона.
2. Не перегружать пользователя математикой.
3. Формулу лучше показать отдельной карточкой внутри popup.
4. Каждый параметр должен быть оформлен отдельным коротким блоком.
5. Popup должен открываться поверх текущего экрана без перехода на новую страницу.
6. Фон за popup затемняется.
7. Закрытие popup:
   - кнопка `Понятно`;
   - нажатие вне popup;
   - свайп вниз, если используется bottom sheet.
8. На мобильном лучше использовать bottom sheet, а не полноэкранный экран.
9. Не использовать casino-style визуал, фишки, карты, рулетку или gambling-графику.
10. Стиль должен соответствовать dark premium UI приложения.

## Acceptance criteria

- В лидерборде рядом с Poker Score есть иконка информации.
- При нажатии открывается popup / bottom sheet.
- Popup содержит понятное описание Poker Score.
- Popup объясняет 4 параметра: ROI Score, Win Rate, Stability Score, Volume Confidence.
- Popup показывает веса параметров: 40%, 30%, 20%, 10%.
- Popup можно закрыть кнопкой `Понятно`.
- Компонент можно переиспользовать в Leaderboard и Player Profile.
- Интерфейс остается удобным на мобильном экране Telegram Mini App.
